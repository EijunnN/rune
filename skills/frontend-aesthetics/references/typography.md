# Typography — 80% of the Design

Swap the typography of any interface and you change what it *is*. It's the highest-
leverage lever and the first thing to get right.

## Choosing fonts with character

The banned-by-default list — not because they're bad, but because they're the
statistical average of generated UI:

- **First-order slop**: Inter, Roboto, Open Sans, Lato, Arial, generic `system-ui`
  everywhere (system stacks are fine for *native-feeling utility apps* — that's a
  direction — not as a non-decision).
- **Second-order slop** — yesterday's "distinctive alternative", now equally
  autogenerated: Space Grotesk on every AI landing page, Poppins on every startup,
  Montserrat headlines, Playfair-on-everything. If a font is the *reflexive* "make it
  designed" pick, it no longer reads as designed.

Instead, pick for the direction (examples, not a new reflex-list):

- **Editorial serif display**: Fraunces, Instrument Serif, Newsreader, Source Serif,
  Libre Caslon, Zodiak-class faces — ink and authority.
- **Characterful grotesque**: General Sans, Cabinet Grotesk, Bricolage Grotesque,
  Archivo (Expanded for headers), Familjen Grotesk, Schibsted Grotesk.
- **Humanist / warm**: Epilogue, Sora, Albert Sans, Work Sans (weights pushed).
- **Mono with taste**: JetBrains Mono, IBM Plex Mono, Fragment Mono, Spline Sans
  Mono, Martian Mono — mono-as-body is a legitimate direction for technical products.
- **Display with a point of view**: Clash Display, Unbounded, Bebas Neue (sparingly),
  Press Start 2P (pixel), Silkscreen, Gloock, Abril Fatface — one of these earns its
  place for headlines only.
- Variable fonts (Fraunces, Bricolage, Archivo) give optical sizes and weight axes —
  one file, many voices.

Verify availability (Google Fonts/Fontshare/self-host) before specifying; never
invent a font name.

## Pairing

Two families is the sweet spot (+ a mono for code):

- **Contrast in one axis, harmony in another**: a high-contrast serif display + a
  neutral grotesque body works because they differ in *style* but match in x-height
  and era. Two similar grotesques differ in nothing and clash in everything.
- Classic recipes: characterful serif display / quiet sans body (editorial);
  loud display sans / same-family body at low weights (modern); mono display / sans
  body (technical); serif body / sans UI-chrome (long-form reading).
- The body font does the work: 400/500 legible at 14–16px, tabular numerals available
  if the UI shows data. The display font takes the risks.

## Scale & hierarchy

- **Build a modular scale** and stick to it — ratio 1.2–1.25 for product UI (dense,
  many levels), 1.333–1.5 for marketing (drama). Define as tokens/utilities, never
  ad-hoc px.
- **Fluid display sizes**: `font-size: clamp(2.5rem, 1.2rem + 4vw, 5rem)` — hero type
  that breathes with the viewport instead of snapping at breakpoints.
- **Weight contrast beats size contrast**: 300 vs 700 reads as designed; 400 vs 500
  reads as accidental. Push to the extremes the family offers.
- Hierarchy is built from *several* signals — size, weight, color (muted vs
  foreground), case, spacing — not size alone. A small, wide-tracked, uppercase,
  muted label above a large tight headline is a stronger hierarchy than two sizes of
  the same style.

## Tuning (where "designed" lives)

- **Tracking**: large display sizes want negative tracking (`-0.02em` to `-0.04em`);
  small uppercase labels want positive (`0.08em`–`0.2em`). Body stays neutral.
  Untracked huge headlines are the most common giveaway of untuned type.
- **Leading**: display 1.0–1.15; body 1.5–1.7; never the same value for both. Long
  measure needs more leading.
- **Measure**: body text 45–75 characters (`max-width: 65ch`). Full-width paragraphs
  are unread paragraphs.
- **Case systems**: uppercase for small labels/eyebrows (with tracking), never for
  body; small-caps via `font-variant-caps` where the face supports it.
- **Numerals**: `font-variant-numeric: tabular-nums` for tables, timers, prices in
  columns — jittering numbers are broken craft.
- **Optical alignment**: pull quotes/bullets into the margin (`text-indent` /
  negative margin) so the *text* edge aligns; center icons against x-height, not
  cap-height. Trust the eye over the box model.
- Ligatures/features: `font-feature-settings` for character alternates when the face
  offers personality there (e.g. single-story a) — one deliberate flourish.

## Loading & implementation

- `next/font` (or equivalent): subset, `display: swap`, preload the display face used
  in the LCP headline; self-host over CDN links; variable font over four static
  weights.
- Expose as CSS variables (`--font-display`, `--font-body`, `--font-mono`) and wire
  into the framework theme so the pairing is swappable in one place.
- Fallback stacks with matching metrics (`size-adjust`/`ascent-override` via next/font
  automatic) to kill layout shift — CLS from fonts is a craft failure.
- Test the real weights you ship: a 300 that renders as synthetic-bold ruins the
  system. Never let the browser fake bold/italic (`font-synthesis: none`).
